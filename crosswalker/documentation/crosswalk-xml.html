The Crosswalk XML File: Defining Crosswalks

Introduction

Please note: this documentation is presently incomplete

The “Crosswalker” toolset relies upon a user-created XML file (the “crosswalk file”) that maps (or describes) the relationships between components of an EpiDoc XML file and components of the targeted format. The structure of the crosswalk XML file is governed by a RelaxNG schema (crosswalk.rng), shipped in the crosswalks directory. Subdirectories within the crosswalks directory may contain example crosswalk xml files (e.g., the experimental iaph2edr-csv, for converting InsAph EpiDoc files to CSV format for input into EDR).

Overview of Document Structure

The crosswalk xml file must be valid in accordance with the crosswalk.rng schema. The top-level element is crosswalk, within which two required elements must appear (in order): input and output. These elements provide essential information about the input and output file formats (see further: the <input> element and the <output> element).

Subsequent to these two elements, any number of copy or force elements may appear, in any order. The simpler of these two elements (force) permits the user to specify a hard-coded value for any “destination” field or element in the output file. The copy element facilitates the replication of data from one or more elements in the source file to a single field (or element) in the output. Multiple source locations (indicated with XPath expressions) and hard-coded default values (if the XPath fails) may be specified. String replacements, value substitutions (for, e.g., vocabulary conversions) and user-supplied XSL transformations may be performed on the data by supplying the appropriate information in the copy.
