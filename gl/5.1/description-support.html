Describing the type of inscribed object (the support)

Release or development version: not supported by cvs2svn

Implementation of the guidance in this section is required for aiegl conformance.

Rigorous epigraphic publications indicate the type of object or monument on which the inscribed text appears. EpiDoc documents must include this information if they aim for EAGLE-conformance.

Editors may include this information in their documents, either as <keyword> <term>s in the <textClass> portion of the <teiHeader> element, or, in the form of prose or text paragraphs, encapsulated in a <div type="description"> element. If an editor wishes to use multiple <div type="description"> elements for different descriptive aspects of the edition, the description element containing the material information should be specifically indicated by using the “subtype” attribute, and giving it a value of “support”.

Whether information about the support is placed in the <teiHeader> or in a <div type="description">, the following tagging examples must be followed closely for EAGLE conformance:

EpiDoc Encoding

  1. Support: <rs type="objectType" key="ara">altar</rs>
  2. <rs type="objectType" key="gradus, theatrum">theater seats </rs>
  3. <rs type="objectType" key="tintinnabulum">bell</rs>

EpiDoc appearance with standard XSLT

  1. Support: altar
  2. theater seats
  3. bell

For EAGLE conformance, the values used in the "key" attribute must be drawn from the EAGLE/EpiDoc Object Type Vocabulary. Values used in the text node of the <rs> element are at the discretion of the editor. Projects are encouraged to control the values in both the key attribute and the text node using DTD or schema modifications (or some other method) to speed data entry, reduce error and ensure uniformity across all documents in a publication or project.
