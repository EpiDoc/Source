List of AIEGL Conformance Requirements

Revised since last release or tag:

The XML encodings laid out in the following sections — in addition to those specified in the Leiden Conformance List — must be applied consistently and completely throughout all transcribed texts in a given digital publication if it is to be considered “AIEGL-conformant EpiDoc”. For fuller explanation, see Conformance (EpiDoc Compatibility).

Note that an XML fragment may be considered a valid EpiDoc file if it is properly handled (e.g., by wrapping an <ab> or <div> element around it and specifying that element as the document root).

  • Describing past and present locations of the inscribed object
  • Describing the type of inscribed object (the support)
  • Describing the material of the inscribed object
  • Measurements
  • Edition
  • Translation
  • Description (or Physical Commentary)
  • History
  • Bibliography
  • Pictorial and other Non-Textual Material
  • Dates
  • EAGLE/EpiDoc Material Vocabulary
  • EAGLE/EpiDoc Object Type Vocabulary
