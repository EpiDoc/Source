Questions and issues

  • An earlier version of the EpiDoc guidelines incorrectly distinguished between these two types of correction, which are not so distinguished in Panciera or other conventions.
  • GB (2006-07-16): I'm going to suggest we quit using the Janus pair of sic/corr elements and attributes, and start using <choice> here (even in P4 land). Not fixed in GL yet because XSLT doesn't yet handle it. See e.g. 3 above.
  • GB (2006-10-25): we now do this in both examples!
  • GB (2008-04-24): deprecated choice/@type, since we only use choice for sic/corr and orig/reg pairs, so there's not much complexity for XSL to deal with.
